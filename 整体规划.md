# 整体规划

## 目标 / 北极星

MacBroom 是一款**开源、本地、零依赖**的 macOS 清理工具，定位是 **CleanMyMac 的「不危险」替代品**。

一句话差异化：**别的清理器先删后道歉，MacBroom 在设计上就不危险**——废纸篓兜底（不 `rm`）、风险分级、危险项默认隐藏、删前 dry-run 确认、全程审计日志、iCloud 感知、受保护路径硬阻断。

成功标准：用户敢把它跑在主力机上而不担心误删；遇到「这工具会不会把我的东西删坏」的疑虑时，MacBroom 是被推荐的那一个。

## 不做什么（边界）

明确排除在 1.1 之外，避免跑偏：

- **App 瘦身 / .lproj 语言包剥离**：高风险低收益——改 App 包内文件可能影响签名/启动，与「不危险」定位有张力。原 TODO-03 已于 2026-06-28 砍掉；如未来要做，归 M3 重新评估，且必须 risky+默认隐藏+实测不破坏启动。
- **磁盘可视化 treemap / Space Lens**：大工程；且已主动去掉 `daisydisk` topic，定位上不主打可视化。
- **重复文件硬链接合并**（保留所有路径同时省空间）：高级且风险高，与「不危险」定位需谨慎权衡。
- **RAM 优化 / 杀资源占用进程**：超出「磁盘清理」范畴。
- **主动卸载在装 App**：竞品在此踩坑最多（误删关联文件，如 Apple Notes / Claude CLI / 微信被误关联）；MacBroom 坚持只清「已卸载 App 的残留」，不做主动卸载，从结构上规避该类事故——这是定位护城河。
- **Homebrew tap 分发**：竞品 brew SHA/cask 坑很多；等 pip 稳定后再议。
- **多语言扩充**（俄/西/日…）：zh/en 已够用，优先级低。
